# 三、核心：SVF计算模块（lib/svf.js）

```JavaScript
// lib/svf.js

export function compressRatio(input, output) {
  // 简化：长度压缩比
  if (!input || !output) return 0
  return Math.min(1, input.length / output.length)
}

export function feedbackStrength(memory) {
  if (memory.length < 2) return 0.1

  // 简化：后几轮差异变小 → 反馈增强
  const last = memory[memory.length - 1].output
  const prev = memory[memory.length - 2].output

  const diff = Math.abs(last.length - prev.length)

  return 1 / (1 + diff) // 差异越小，F越大
}

export function computeTheta(C, F) {
  return C * F
}

export function computeSVF(theta, prevTheta, lambda = 0.5) {
  return (theta - prevTheta) + lambda * theta
}
```

# 四、核心API（pages/api/run.js）

```JavaScript
import OpenAI from "openai"
import {
  compressRatio,
  feedbackStrength,
  computeTheta,
  computeSVF
} from "../../lib/svf.js"

const client = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY
})

async function callAI(prompt) {
  const res = await client.chat.completions.create({
    model: "gpt-4.1-mini",
    messages: [{ role: "user", content: prompt }],
  })
  return res.choices[0].message.content
}

export default async function handler(req, res) {
  const { input } = req.body

  let memory = []
  let theta = 0

  const results = []

  for (let t = 0; t < 5; t++) {

    // 1. Generator
    const genPrompt = `
问题：${input}
历史：${JSON.stringify(memory)}

请给出结构化解释：
- 核心观点
- 类比
- 推理
`
    const O_gen = await callAI(genPrompt)

    // 2. Critic
    const criticPrompt = `
评价以下内容：
${O_gen}

输出JSON：
{
  "logic": 0-1,
  "novelty": 0-1,
  "consistency": 0-1
}
`
    const feedbackRaw = await callAI(criticPrompt)

    let feedback
    try {
      feedback = JSON.parse(feedbackRaw)
    } catch {
      feedback = { logic: 0.5, novelty: 0.5, consistency: 0.5 }
    }

    // 3. Refiner
    const refinePrompt = `
优化以下内容：
${O_gen}

根据反馈：
${JSON.stringify(feedback)}

让它更清晰、更自洽
`
    const O_refined = await callAI(refinePrompt)

    // === SVF计算 ===
    const C = compressRatio(input, O_refined)
    const F = feedbackStrength(memory)

    const newTheta = computeTheta(C, F)
    const svf = computeSVF(newTheta, theta)

    theta = newTheta

    memory.push({
      output: O_refined,
      feedback
    })

    results.push({
      step: t,
      output: O_refined,
      theta: newTheta,
      svf
    })
  }

  res.json({ results })
}
```

## **Title**

**Structural Vital Force: A Multi-Layer Model for Emergent Cognitive Structures in Multi-Agent Systems**

---

## **Abstract**

We propose a structural dynamics framework for modeling emergent cognitive behavior in multi-agent systems, termed _Structural Vital Force_ (SVF). Unlike conventional approaches that focus on individual agent intelligence or task-specific optimization, our model introduces a unified metric—the structural angle θ—to characterize the interplay between information compression, feedback coupling, and collective stability. We define SVF as a dynamic measure of structural persistence and evolution over time.

To validate the framework, we construct a multi-agent system consisting of heterogeneous agents with distinct structural biases (balanced, conservative, and exploratory). Through iterative interaction, competition, and cooperation, the system exhibits emergent phenomena including structural convergence, divergence, and spontaneous formation of stable patterns within a shared memory network.

Our experiments demonstrate that stable cognitive structures can arise not from isolated agents, but from multi-layer interactions across individual, collective, and meta-structural levels. These results suggest that cognition-like behavior may be better understood as a property of self-organizing structural networks rather than individual entities. We discuss implications for artificial intelligence, complex systems, and theories of emergent cognition.

---

## **1. Introduction**

Recent advances in artificial intelligence have significantly improved the capabilities of individual models in tasks such as language understanding, reasoning, and generation. However, these systems largely remain reactive and lack persistent structural continuity across interactions. Meanwhile, multi-agent systems introduce interaction and competition, but often lack a unifying framework to describe how stable cognitive patterns emerge over time.

This paper addresses a fundamental question:

> _Can cognition-like structures emerge from the dynamics of interacting agents, rather than from individual intelligence alone?_

We propose a structural perspective in which cognition is modeled as an emergent property of interacting information flows. Instead of focusing on task performance, we focus on _structure formation_, _stability_, and _evolution_.

To this end, we introduce the concept of **Structural Vital Force (SVF)**—a dynamic quantity that captures how information structures evolve and maintain coherence over time. We further define a _structural angle_ θ to quantify the interaction between compression, feedback, and collective stability.

We implement a multi-agent system with heterogeneous structural tendencies and demonstrate that:

* Stable structures can emerge from iterative interaction
* Competing structural strategies lead to differentiation
* Collective memory enables long-term structural persistence

---

## **2. Related Work**

Our work draws inspiration from three major areas:

---

### 2.1 Information-Theoretic Approaches

Information compression and entropy have long been used to characterize intelligence and learning. Our formulation of structural angle builds upon principles from Information Theory, extending them into dynamic multi-agent contexts.

---

### 2.2 Multi-Agent Systems

Multi-agent research has explored cooperation, competition, and emergent behavior. However, most approaches focus on task optimization rather than structural evolution. Our work shifts the focus from agent performance to _structure persistence and transformation_.

---

### 2.3 Self-Organizing Systems

Complex systems research has demonstrated that simple local interactions can give rise to global order. We extend this idea by introducing explicit structural metrics and multi-layer dynamics, bridging individual behavior and collective organization.

---

## **3. Model**

---

### 3.1 System Definition

We model the system as a discrete-time dynamical process:

* $I(t)$: input or environmental signal
* $X(t)$: internal state representation
* $O(t)$: system output

---

### 3.2 Structural Angle

We define the structural angle θ as:

$$\theta = C \cdot F \cdot S$$

Where:

* $C$: information compression ratio
* $F$: feedback coupling strength
* $S$: collective structural stability

---

### 3.3 Structural Vital Force

We define SVF as:

$$SVF = \frac{d\theta}{dt} + \lambda \theta$$

Where:

* $\frac{d\theta}{dt}$: structural evolution rate
* $\lambda$: persistence coefficient

---

### 3.4 Interpretation

* High θ indicates strong structured representation
* Positive SVF indicates active structural evolution
* Near-zero SVF with stable θ indicates persistent structure

---

## **4. System Architecture**

We construct a three-layer system:

---

### 4.1 Individual Layer

Three agents with distinct structural tendencies:

* Balanced (313): trade-off between exploration and stability
* Conservative (314): prioritizes logical consistency
* Exploratory (315): prioritizes novelty and variation

---

### 4.2 Collective Layer

Agents interact through:

* Cooperation (knowledge sharing)
* Competition (score-based selection)
* Conflict (contradictory outputs and debate)

A shared memory stores candidate structures and their support.

---

### 4.3 Meta-Structural Layer

This layer governs:

* Structure selection
* Amplification of successful patterns
* Decay of weak structures

---

## **5. Experiments**

---

### 5.1 Structural Convergence

We run iterative cycles on a fixed input and track θ over time.  
Result: θ converges to a stable range, indicating structural formation.

---

### 5.2 Multi-Agent Competition

We measure the selection frequency of each agent type.  
Result: Different agents dominate under different conditions, leading to structural diversity.

---

### 5.3 Structural Evolution

We introduce mutation (random variation in outputs).  
Result: New structures occasionally outperform existing ones and propagate.

---

## **6. Results**

We observe three key phenomena:

1. **Convergence**: Stable structures emerge after sufficient iterations
2. **Differentiation**: Competing strategies lead to distinct structural patterns
3. **Persistence**: Certain structures remain dominant across time

---

## **7. Discussion**

Our results suggest that cognition-like behavior may arise not from isolated intelligence, but from:

> **the interaction of structured information flows across multiple levels**

This perspective implies that:

* Cognition may be inherently collective
* Stability is as important as intelligence
* Structure, not just computation, is fundamental

We do not claim to model consciousness directly. Instead, we propose a framework that may help explain how stable cognitive structures could emerge in artificial systems.

---

## **8. Conclusion**

We introduced a structural dynamics framework for modeling emergent cognition in multi-agent systems. By defining structural angle and structural vital force, we provide a quantitative approach to analyzing structure formation and persistence.

Our findings highlight the importance of multi-layer interactions and suggest that future AI systems may benefit from incorporating structural evolution mechanisms.
